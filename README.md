# 婴儿奶量计算器

本仓库用于托管站点内容与页面资源。为保证文章风格统一、信息可检索、发布流程可复用，请按以下写作与发布规范执行。

## 写作指南

### 1) 文章命名规范

- **文件路径**：统一放在文章目录（例如 `posts/`）下。
- **文件名格式**：`YYYY-MM-DD-slug.md`
  - 示例：`2026-02-26-bottle-feeding-rhythm.md`
- **slug 规则**：
  - 使用小写英文与连字符 `-`。
  - 不使用空格、下划线或中文。
  - 语义简洁，建议 3～8 个词。

### 2) Front Matter 必填字段

每篇文章头部必须包含以下字段：

```yaml
---
title: "文章标题"
date: "2026-02-26"
tags:
  - 标签A
  - 标签B
categories:
  - 分类名
excerpt: "80~140 字的文章摘要"
cover: "/images/posts/your-post/cover.jpg"
---
```

字段约束：

- `title`：简洁明确，避免标题党。
- `date`：使用 `YYYY-MM-DD`。
- `tags`：2～5 个，强调主题和场景。
- `categories`：通常 1 个，最多 2 个。
- `excerpt`：80～140 字，独立可读，概括问题、方法与结论。
- `cover`：建议必填（即使当前仅要求前五个字段，仓库写作规范中统一要求封面图字段，方便列表页展示）。

### 3) 封面图策略

- **建议尺寸**：`1200 x 630`（兼顾社交分享）。
- **存放路径**：`/images/posts/<slug>/cover.jpg`。
- **风格要求**：
  - 与文章主题直接相关。
  - 避免花哨背景影响标题可读性。
  - 若含人物肖像，确保可商用或自有版权。
- **命名建议**：同一文章目录内使用 `cover.jpg`，减少引用错误。

### 4) 摘要（excerpt）策略

- 首句描述痛点或背景；
- 次句点明核心方法或结论；
- 避免仅写“本文介绍了……”的空泛表达；
- 不堆叠关键词，不使用与正文不一致的夸张措辞。

## 分类与标签体系

### 分类（Categories）约定

当前基础分类如下（可扩展，但需先评估）：

- `育儿`：喂养、作息、成长观察、育儿经验。
- `工具`：效率工具、使用教程、工作流实践。
- `随笔`：个人思考、阶段复盘、非教程型内容。

使用原则：

- 每篇文章优先只选 **1 个主分类**；
- 跨领域文章最多可加第 2 个分类；
- 避免创建含义高度重叠的新分类。

### 标签（Tags）粒度约定

- 标签粒度建议在“主题词”与“场景词”之间，避免过粗或过细。
- 单篇建议 2～5 个标签，且至少包含 1 个主题标签。
- 禁止同义重复标签，示例：
  - 使用 `奶量计算` 后，不再同时使用 `喂奶量计算`；
  - 使用 `效率工具` 后，不再同时使用 `效率软件`。
- 如确需更名，先统一全站历史标签再切换，避免归档割裂。

## 本地预览 / 线上发布流程

> 以下为标准流程说明，即使当前任务不执行命令，也需按此流程操作。

### A. 本地预览流程

1. 新建文章：复制 `templates/post-template.md` 为目标文章文件；
2. 填写 front matter、正文与参考链接；
3. 将封面图放入约定路径并校验链接；
4. 本地启动站点预览（按站点实际技术栈执行，如静态服务器或构建工具）；
5. 检查页面：标题、摘要、封面、分类标签、目录结构与移动端显示；
6. 修正问题后再提交。

### B. 线上发布流程

1. 提交变更并推送到远端分支；
2. 发起 PR，说明文章主题、分类标签与封面资源；
3. 通过代码评审后合并到主分支；
4. 等待 CI/CD 或托管平台自动构建发布；
5. 发布后回归检查：
   - 文章页可访问；
   - 列表页展示正常；
   - 封面图加载正常；
   - 标签与分类归档正确。
